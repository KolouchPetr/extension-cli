Getting Started

Before you start, be aware that your project is expected to have a certain file structure.

You can override most of these paths when specifying build and documentation settings, but sticking to the default organization will make your life easier so we highly recommend its usage.

Eventually we will add a command that will set up this structure automatically.

Prerequisites

Before using extension CLI, you should have all of the following:

  • Node.js
  • Any web-friendly IDE
  • Terminal/cmd access
  • Chrome browser for debugging extensions

Installation

Install latest CLI version from NPM

npm install extension-cli

Basic Configuration

Update extension project package.json with following options:

  "babel": {
    "presets": [
      "@babel/preset-env"
    ]
  }
Babel presets

Needed to compile projects written in ES6. 
  "eslintIgnore": [
    "test/**/*"
  ]
ESLint ignore

Exclude test files from being linted. If your project includes compiled 3rd party libraries, you should exclude them also. 
  "xtdocs": {
    "templates": {
      "systemName": "Extension name",
      "systemSummary": 
        "Write a short description here!",
      "systemColor": "#000000"
    }
  }
